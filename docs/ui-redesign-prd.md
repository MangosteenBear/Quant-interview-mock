# 量化面试题库 — UI 重设计 PRD

> 版本 v1.0 | 2026-06-29 | 参考标杆：LeetCode、小站雅思

---

## 一、背景与问题

### 1.1 现状问题

| 页面 | 核心问题 |
|------|---------|
| 首页 | 只有题目总数 + 4 个快捷按钮，缺乏用户激励和学习感 |
| 列表页 | 无做题状态（已做/未做），无排序，筛选交互笨重 |
| 详情页 | 无上下题导航，无做题计时展示，解析交互弱 |
| 全局 | 无做题进度/完成率，缺成就感，无留存动力 |

### 1.2 目标用户

量化求职者（本科/研究生），备战 Jane Street、Citadel、Two Sigma、Jump Trading 等量化岗面试，主要使用场景：**地铁/课间碎片时间刷题**，需要移动端优先的轻量体验。

## 二、设计原则（参考 LeetCode + 小站雅思）

| 原则 | LeetCode 做法 | 小站雅思做法 | 本项目应用 |
|------|--------------|------------|----------|
| **进度可见** | 每道题有已做/通过状态，主页显示完成率 | 每个模块显示进度条 | 列表题目状态标记 + 首页完成率环形图 |
| **分层导航** | 按标签/公司/难度多维筛选 | 按词汇/阅读/写作分模块 | 按知识点/题型/难度三维 |
| **激励机制** | 连续打卡 streak、随机一题 | 每日任务、练习计划 | 每日推荐题 + 做题数统计 |
| **详情沉浸** | 上下题导航不离开，相关题目推荐 | 当前进度 N/总数 | 详情页内翻题 + 进度指示 |
| **搜索增强** | 模糊搜索 + 历史记录 | 高亮匹配词 | 历史搜索词 + 结果高亮 |

## 三、首页重设计

### 3.1 功能模块

```
┌─────────────────────────────────┐
│  打招呼区（今日日期 + 激励语）      │
├─────────────────────────────────┤
│  学习进度卡                      │
│  ┌──────────┐  ┌──────────────┐│
│  │ 环形进度图 │  │ 已做 / 总题数 ││
│  │ 完成率 %  │  │ 连续打卡 N 天 ││
│  └──────────┘  └──────────────┘│
├─────────────────────────────────┤
│  每日推荐题（1-2 道，按历史偏好）  │
│  [难度标签] [题型标签] 题干预览   │
│  ► 开始练习                      │
├─────────────────────────────────┤
│  分类入口 （2×2 网格）           │
│  ┌───────────┐ ┌───────────┐   │
│  │ 概率论     │ │ 随机过程   │   │
│  │ 34 题      │ │ 18 题     │   │
│  └───────────┘ └───────────┘   │
│  ┌───────────┐ ┌───────────┐   │
│  │ 期权定价   │ │ 统计/线代  │   │
│  │ 12 题      │ │ 21 题     │   │
│  └───────────┘ └───────────┘   │
├─────────────────────────────────┤
│  快捷操作（一行）                 │
│  🔍 搜索  ⭐ 收藏  🎲 随机一题   │
└─────────────────────────────────┘
```

### 3.2 数据需求

| 数据 | 来源 | 说明 |
|------|------|------|
| 完成率 | 前端 `attempt_logs`（本地 device_id） | 后端已有 attempt_logs 表 |
| 连续打卡 | 前端 localStorage 记录每日首次作答日期 | 纯前端实现，无需后端 |
| 每日推荐题 | 按日期 seed + 未做题目随机选取 | 前端逻辑 |
| 按知识点分类题数 | `GET /api/tags` + `GET /api/questions?tag_id=N` | 后端已有标签体系 |

## 四、题目列表页重设计

### 4.1 功能变更

**新增：**
- 题目状态标记（未做 / 已做 / 答对 ✓ / 答错 ✗）
- 排序切换（默认 / 难度升降 / 最新）
- 侧边标签栏（知识点快速切换）
- 题数统计行（"显示 15 道题 · 已做 3 道"）

**优化：**
- 筛选 chip 改为水平滚动标签组（不遮挡内容）
- QuestionCard 增加"已做"状态角标

### 4.2 列表项新设计

```
┌──────────────────────────────────────┐
│ ✓  两枚骰子点数之和为 7 的概率...       │  ← 状态图标
│    概率论  选择题  ● ● ○ ○ ○ P1      │  ← 难度点阵
│    Heard on the Street · P12        │  ← 来源/页码
└──────────────────────────────────────┘
```

## 五、题目详情页重设计

### 5.1 功能新增

| 功能 | 优先级 | 实现方式 |
|------|--------|---------|
| 上下题导航（列表内翻题） | P0 | store 中维护 currentIndex，底部 ← → 按钮 |

| 做题计时展示 | P1 | 提交后显示"用时 X 秒" |
| 解析默认展开（已做状态复访） | P1 | 检测 submitted 状态 |
| 相关知识点标签（可点击） | P2 | 点击跳列表页并筛选该标签 |

### 5.2 页面结构调整

```
顶部：[← 返回] [题目 3/15]             [★ 收藏]
──────────────────────────────────────

──────────────────────────────────────

──────────────────────────────────────

解析区（折叠 → 默认收起，提交后展开）

──────────────────────────────────────
底部导航：[← 上一题]          [下一题 →]
```

## 八、全局 UI 规范调整

### 8.1 颜色体系

| 用途 | 当前值 | 建议 |
|------|--------|------|
| 主色 | `#1e3a5f`（深蓝） | 保留，品牌色 |
| 成功/答对 | `#d4edda` | `#eaf3de`（更轻） |
| 错误/答错 | `#f8d7da` | `#fcebeb`（更轻） |
| 难度 P1 | 绿色 | 保留渐变 P1→P5 绿→红 |
| 已做状态 | 无 | 新增 `#5DCAA5`（teal）表示完成 |
| 未做状态 | 无 | `#888780`（gray）表示未做 |

### 8.2 Tab Bar 调整

| 当前 Tab | 调整后 |
|---------|--------|
| 首页 | 首页（不变） |
| 题库（列表） | 题库（不变） |
| 收藏 | 收藏（不变） |
| 设置 | 统计（新增，替换设置） |
| — | 设置移到首页右上角齿轮 |

### 8.3 字体与间距

- 题干字体：16px，行高 1.7（当前偏小，建议调整）
- 卡片内边距：16px → 统一 `var(--pad-md)`
- 难度标签：点阵式（● ● ○ ○ ○）替代文字标签（P1），更直观

## 九、实施优先级

| 优先级 | 内容 | 工作量 | 依赖 |
|--------|------|--------|------|
| **P0** | 详情页上下题导航 + 进度指示 | 1 天 | 无 |
| **P0** | 列表页题目状态标记（已做/答对/答错） | 1.5 天 | attempt_logs 读取 |
| **P0** | 首页进度卡（完成率 + 已做数） | 1 天 | 本地 attempt 数据 |
| **P1** | 首页分类入口（按知识点标签） | 1 天 | `GET /api/tags` |
| **P1** | 搜索历史 + 结果分页 | 1 天 | 无 |
| **P1** | 每日推荐题 | 0.5 天 | 无 |
| **P2** | 学习统计页（热力图 + 题型完成率） | 2 天 | 本地数据 |
| **P2** | 随机一题按钮 | 0.5 天 | 无 |
| **P3** | Tab Bar 重组（统计 Tab） | 0.5 天 | 统计页完成后 |
